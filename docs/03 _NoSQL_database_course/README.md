# 03 — NoSQL Database Course: MongoDB · Redis · Neo4j · DynamoDB

## **Objetivo**

`Dominar NoSQL por **modelos y tecnologías**, entendiendo:`

- Qué problema resuelve cada familia NoSQL.
- Cómo cambia el diseño respecto a relacional.
- Patrones de modelado por tecnología.
- Operación mínima: rendimiento, escalado, seguridad, backup/restore.
- Construcción de criterio: elegir tecnología por caso de uso.

## **Indicadores de dominio (NoSQL listo “para mundo real”)**

- Puedes justificar por qué un caso es mejor en documentos vs grafos vs clave-valor.
- Diseñas un modelo NoSQL alineado a patrones de consulta.
- Manejas índices/capacidad/rendimiento a nivel básico-intermedio.
- Entiendes riesgos: consistencia, escalado, y tradeoffs frente a SQL.
