# CURSO DE BASES DE DATOS- MÓDULO 2 **DISEÑO Y MODELADO DE BASES DE DATOS**

## MODÉLO LÓGICO DE BASES DE DATOS

### ATRIBUTOS

Cada atributo de una entidad, representa a un campo de una tabla de nuestra base de datos.

Cuando es compuesto, se recomienda poner _ en lugar de espacios.

Por ejemplo

![alt text](A3.PNG)

`En el formato gráfico, los atributos se ponen entre parentesis dentro de cada entidad.`

Los atributos identificativos se subrayan con una linea negra.

Si es un atributo compuesto, solo se ponen los atributos hijos, no el padrre (en este caso, dirección).

Los atributos multivaluados se colocan como si fuera un atributo mas.

Los atributos calculados NO se representan.

Los atributos que puedan ser nulos, se representan con un * al lado del nombre

Empleado (DNI, nombre, email, ciudad, calle, numero, telefono, fecha_nacimiento, comision*)

En el formato de tabla, cada fila representará a cada atributo con su respectiva información.

En la columna restricciones, indicaremos si tienen algun tipo de restricción como que es atributo identificativo, unico, etc.

Por defecto, entenderemos que los campos no pueden ser nulos, si alguno lo es, debes indicarlo.
