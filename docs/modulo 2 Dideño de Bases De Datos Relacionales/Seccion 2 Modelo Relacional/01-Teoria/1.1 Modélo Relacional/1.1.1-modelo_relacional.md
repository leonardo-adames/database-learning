## 2.2  INTRODUCCIÓN AL MODÉLO RELACIONAL DE BASES DE DATOS

### DEFINICIÓN

* El modelo relacional es un enfoque para organizar datos en tablas (`llamadas relaciones`) compuestas por filas (`tuplas`) y columnas (`atributos`), basado en la teoría de conjuntos y la lógica matemática, propuesto por E.F. Codd. Su objetivo es separar la vista lógica de los datos de su almacenamiento físico, permitiendo flexibilidad y eficiencia, y es la base de los sistemas de gestión de bases de datos (`DBMS`) más populares, como `MySQL` y `Oracle`, usando `SQL` para manipularlos.

### Conceptos Clave

* **Relación (Tabla):** Una colección de datos sobre una entidad, organizada en filas y columnas.

* **Atributo (Columna):** Una característica o propiedad de la entidad, con un nombre y un tipo de dato (dominio).

* **Tupla (Fila):** Un registro individual dentro de una tabla, que contiene los valores de los atributos para una instancia específica.

* **Clave Primaria (Primary Key):** Un atributo o conjunto de atributos que identifica de forma única a cada tupla en una relación (ej. un ID de cliente).

* **Clave Foránea (Foreign Key):** Un atributo en una tabla que referencia la clave primaria de otra tabla, estableciendo un vínculo entre ellas (`integridad referencial`).

### Características Principales

* **Independencia de Datos:** El usuario no necesita saber cómo se almacenan físicamente los datos; solo interactúa con la estructura lógica (tablas).

* **Integridad de Datos:** Reglas (dominio, entidad, referencial) para asegurar la precisión y consistencia de la información.
* **Normalización:** Proceso para reducir redundancia y mejorar la organización de los datos.

* **Manipulación flexible:** Se accede y modifica mediante lenguajes de consulta estructurados como `SQL`, que ofrecen gran poder.

### ¿Para qué sirve?

* Es el modelo más usado para gestionar datos estructurados, permitiendo diseñar bases de datos robustas, seguras y fáciles de consultar, desde simples aplicaciones hasta grandes sistemas empresariales.
