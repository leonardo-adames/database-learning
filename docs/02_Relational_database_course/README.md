# 02 — Curso Relacional: SQL (MySQL) + Práctica

## Objetivo
Convertir el diseño relacional (ER → Relacional → Normalización) en una implementación real con **SQL en MySQL**, desarrollando dominio en:

- DDL: creación de tablas, llaves, constraints.
- DML: inserciones, actualizaciones, borrado.
- Consultas: filtros, joins, agregaciones, subconsultas.
- Integridad, transacciones y nociones de rendimiento.
- Práctica sistemática con ejercicios y tests (SQL Vault) + modelado/visualización del esquema (ChartDB).

## Indicadores de dominio (listo para pasar a NoSQL)

`Puedes pasar a NoSQL cuando:`

- Creas un esquema completo en MySQL con PK/FK correctas.
- Escribes JOINS correctamente para responder preguntas reales.
- Dominas agregaciones (SUM/COUNT/AVG/MIN/MAX) y GROUP BY.
- Sabes cuándo usar subconsultas vs joins.
- Entiendes (a nivel básico) qué problema resuelven índices y transacciones.
