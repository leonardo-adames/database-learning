# CURSO DE BASES DE DATOS- MÓDULO 2 **MER [Modélo Entidad / Realción]**

## 2.3 Modélo Entidad-Realción EXTENDIDO

## **Definición:**

Es una extensión del modelo ER básico que añade conceptos de la programación orientada a objetos, como jerarquías de herencia (`super clases` / `sub clases` / `Subtipos` / `Entidades Hijas`), `especialización` y `generalización`, y `categorías / uniones`, para representar estructuras de datos más complejas y abstractas con mayor expresividad, permitiendo modelar refinamientos detallados de entidades y sus relaciones, como empleados que son secretarias, ingenieros o gerentes, y cómo comparten o difieren atributos y comportamientos.

## **Propósito:**

* Aumentar la capacidad expresiva del modelo, permitiendo representar de forma más detallada la realidad y las relaciones complejas dentro de un sistema de información.

* Facilitar el diseño de bases de datos robustas y flexibles, anticipando necesidades futuras de organización de datos. En resumen, el `modelo Entidad-Relación Extendido` toma el lenguaje gráfico del `modelo Entidad-Relación` y le añade la potencia de la `herencia`, `Encapculación`, `Poliformísmo O jerarquía` y la clasificación para modelar sistemas complejos con mayor claridad y precisión, acercándose a paradigmas orientados a objetos.
